### Task 1: Fix `stop()` — send `stop` not `quit`

**Files:**

- Modify: `src/index.ts:188-190`
- Test: `src/__tests__/index.spec.ts`

**Background:** `stop()` currently sends `quit` (kills the engine process). It
should send `stop` (halts search, keeps engine alive). `[Symbol.dispose]()` is
the correct place to send `quit` and terminate the process.

**Step 1: Write the failing test**

Add to `src/__tests__/index.spec.ts` inside `describe('UCI', ...)`:

```typescript
it('stop() does not kill the engine process', async () => {
  // We can't easily test process liveness without a real engine,
  // so we test that stop() sends 'stop' by spying on execute.
  const uci = new UCI('/invalid/path');
  const spy = vi.spyOn(
    uci as unknown as { execute: (cmd: string) => Promise<void> },
    'execute',
  );

  // stop() should not throw — ignore the underlying write error
  await uci.stop().catch(() => {});

  expect(spy).toHaveBeenCalledWith('stop');
});

it('[Symbol.dispose]() sends quit', async () => {
  const uci = new UCI('/invalid/path');
  const spy = vi.spyOn(
    uci as unknown as { execute: (cmd: string) => Promise<void> },
    'execute',
  );

  await uci[Symbol.dispose]().catch(() => {});

  expect(spy).toHaveBeenCalledWith('quit');
});
```

**Step 2: Run tests to verify they fail**

```bash
pnpm test src/__tests__/index.spec.ts
```

Expected: FAIL — `stop()` called with `'quit'`, `Symbol.dispose` does not exist.

**Step 3: Fix `stop()` and add `[Symbol.dispose]()`**

In `src/index.ts`, replace:

```typescript
stop(): Promise<void> {
  return this.execute('quit');
}
```

With:

```typescript
async [Symbol.dispose](): Promise<void> {
  await this.execute('quit');
  this.process.kill();
}

stop(): Promise<void> {
  return this.execute('stop');
}
```

Keep the natural ordering: `[Symbol.dispose]` is a public method, place it
before `stop()` (alphabetical among public methods).

**Step 4: Run tests to verify they pass**

```bash

```

Expected: PASS (plus the existing error emission test).

### Task 2: Fix `extract` to support repeated keys

**Files:**

- Modify: `src/parser/extract.ts`

**Background:** `extract` collapses repeated keys into a single space-joined
string. For `combo` options, `var` appears once per variant, so the result must
be `string[]`. The fix: when a key is seen more than once, its value becomes (or
appends to) a `string[]`.

**Step 3: Update `extract` to collect repeated keys as arrays**

Replace `src/parser/extract.ts` with:

```typescript
function extract<K extends string>(
  keys: K[],
): (value: string) => Partial<Record<K | 'default', string | string[]>> {
  return (value: string) => {
    const chunks = value.split(' ');
    const extraction: Partial<Record<K | 'default', string | string[]>> = {};

    let key: K | 'default' = 'default';
    for (const chunk of chunks) {
      if (keys.includes(chunk as K)) {
        key = chunk as K;
        if (extraction[key] === undefined) {
          extraction[key] = '';
        }
      } else {
        const current = extraction[key];
        if (current === undefined) {
          extraction[key] = chunk;
        } else if (Array.isArray(current)) {
          // Key has already appeared multiple times — append to array
          const last = current.at(-1) ?? '';
          current[current.length - 1] = last
            ? `${last} ${chunk}`.trim()
            : chunk;
        } else {
          // First token after this key occurrence
          const appended = current ? `${current} ${chunk}`.trim() : chunk;
          extraction[key] = appended;
        }
      }
    }

    return extraction;
  };
}

export default extract;
```

Wait — re-read the intent. Each `var Solid`, `var Normal`, `var Risky` means the
key `var` appears 3 times; each occurrence should produce one entry. The logic
should be: when a key token is seen again (already has a value), convert to
array and start a new entry.

Replace `src/parser/extract.ts` with:

```typescript
function extract<K extends string>(
  keys: K[],
): (value: string) => Partial<Record<K | 'default', string | string[]>> {
  return (value: string) => {
    const chunks = value.split(' ');
    const extraction: Partial<Record<K | 'default', string | string[]>> = {};

    let key: K | 'default' = 'default';
    for (const chunk of chunks) {
      if (keys.includes(chunk as K)) {
        key = chunk as K;
        const current = extraction[key];
        if (current === undefined) {
          // First occurrence — initialise as empty string sentinel
          extraction[key] = '';
        } else if (Array.isArray(current)) {
          // Third+ occurrence — push a new empty-string slot
          current.push('');
        } else {
          // Second occurrence — convert to array
          extraction[key] = [current, ''];
        }
      } else {
        const current = extraction[key];
        if (current === undefined) {
          extraction[key] = chunk;
        } else if (Array.isArray(current)) {
          const last = current.at(-1) ?? '';
          current[current.length - 1] = last
            ? `${last} ${chunk}`.trim()
            : chunk;
        } else {
          extraction[key] = current ? `${current} ${chunk}`.trim() : chunk;
        }
      }
    }

    // Trim any trailing empty-string slots introduced by key-only tokens
    for (const k of Object.keys(extraction) as (K | 'default')[]) {
      const v = extraction[k];
      if (Array.isArray(v)) {
        extraction[k] = v.map((s) => s.trim()).filter((s) => s.length > 0);
      } else if (typeof v === 'string') {
        extraction[k] = v.trim();
      }
    }

    return extraction;
  };
}

export default extract;
```

### Task 3: Fix `lowerbound`/`upperbound` parsing in `parseScore`

**Files:**

- Modify: `src/parser/info.ts`
- Test: `src/__tests__/parser.spec.ts`

**Background:** `lowerbound` and `upperbound` are standalone flag tokens in the
`score` segment (e.g. `score cp 50 upperbound`). They have no value. Currently
they are treated as `extract` keys, which is order-dependent. The fix: remove
them from the extractor and detect them as flags within `parseScore` by scanning
the raw `score` token string.

**Step 1: Verify existing bound tests pass (they should), then add an
order-independence test**

```typescript
    score: { bound: 'lower', type: 'cp', value: -0.3 },
  });
});
```

**Step 2: Run test to verify it fails**

```bash
pnpm test src/__tests__/parser.spec.ts -- -t "lowerbound score regardless"
```

Expected: FAIL — bound is not detected when `lowerbound` appears before `cp`.

**Step 3: Update `info.ts`**

Remove `lowerbound` and `upperbound` from the extractor field list, remove them
from the destructuring, and update `parseScore` to detect them as flags within
the raw score string:

```typescript
import extract from './extract.js';

import type { InfoCommand, Score } from '../types.js';

const extractor = extract([
  'cpuload',
  'currline',
  'currmove',
  'currmovenumber',
  'depth',
  'hashfull',
  'multipv',
  'nodes',
  'nps',
  'pv',
  'refutation',
  'score',
  'seldepth',
  'string',
  'tbhits',
  'time',
]);

function parseScore(raw: string): Score | undefined {
  const tokens = raw.trim().split(/\s+/);
  const hasLowerbound = tokens.includes('lowerbound');
  const hasUpperbound = tokens.includes('upperbound');
  const remaining = tokens.filter(
    (t) => t !== 'lowerbound' && t !== 'upperbound',
  );

  const kind = remaining[0];

  if (kind !== 'cp' && kind !== 'mate') {
    return undefined;
  }

  const rawValue = remaining[1];

  if (rawValue === undefined) {
    return undefined;
  }

  const numeric = Number(rawValue);

  if (!Number.isFinite(numeric)) {
    return undefined;
  }

  const value = kind === 'cp' ? numeric / 100 : numeric;

  if (kind === 'cp') {
    if (hasLowerbound) {
      return { bound: 'lower', type: 'cp', value };
    }

    if (hasUpperbound) {
      return { bound: 'upper', type: 'cp', value };
    }

    return { type: 'cp', value };
  }

  return { type: 'mate', value };
}

function info(value: string): InfoCommand {
  const {
    cpuload,
    currline,
    currmove,
    currmovenumber,
    depth,
    hashfull,
    multipv,
    nodes,
    nps,
    pv,
    refutation,
    score,
    seldepth,
    string,
    tbhits,
    time,
  } = extractor(value);

  const parsedScore =
    score === undefined || Array.isArray(score) ? undefined : parseScore(score);

  return {
    ...(cpuload !== undefined &&
      !Array.isArray(cpuload) && { cpuload: Number(cpuload) }),
    ...((currline !== undefined ||
      currmove !== undefined ||
      currmovenumber !== undefined) && {
      current: {
        ...(currline !== undefined && {
          line: (Array.isArray(currline) ? currline.join(' ') : currline)
            .trim()
            .split(' '),
        }),
        ...(currmove !== undefined &&
          !Array.isArray(currmove) && { move: currmove }),
        ...(currmovenumber !== undefined &&
          !Array.isArray(currmovenumber) && { number: Number(currmovenumber) }),
      },
    }),
    ...(depth !== undefined &&
      !Array.isArray(depth) && {
        depth:
          seldepth === undefined || Array.isArray(seldepth)
            ? Number(depth)
            : { selective: Number(seldepth), total: Number(depth) },
      }),
    ...(hashfull !== undefined &&
      !Array.isArray(hashfull) && { hashfull: Number(hashfull) }),
    ...(string !== undefined && !Array.isArray(string) && { info: string }),
    ...(multipv !== undefined &&
      !Array.isArray(multipv) && { line: Number(multipv) }),
    ...(pv !== undefined && {
      moves: (Array.isArray(pv) ? pv.join(' ') : pv).trim().split(' '),
    }),
    ...(nodes !== undefined &&
      !Array.isArray(nodes) && { nodes: Number(nodes) }),
    ...(nps !== undefined &&
      !Array.isArray(nps) && { stats: { nps: Number(nps) } }),
    ...(refutation !== undefined && {
      refutation: (Array.isArray(refutation)
        ? refutation.join(' ')
        : refutation
      )
        .trim()
        .split(' '),
    }),
    ...(parsedScore !== undefined && { score: parsedScore }),
    ...(tbhits !== undefined &&
      !Array.isArray(tbhits) && { tbhits: Number(tbhits) }),
    ...(time !== undefined && !Array.isArray(time) && { time: Number(time) }),
  };
}

export default info;
```

Note: since `extract` now returns `string | string[]`, all field accesses need
`Array.isArray` guards. For `info` fields that are never repeated (all of them
except potentially none), these guards are always false — but TypeScript
requires them.

### Task 4: Fix `position` setter — propagate write errors

**Files:**

- Modify: `src/index.ts:129`
- Test: `src/__tests__/index.spec.ts`

**Background:** The `position` setter calls `this.execute(...)` via `.then()`
with no `.catch()`. Any write failure is silently swallowed. The fix is a
one-liner `.catch()` using the same pattern as `execute()` itself.

**Step 1: Write the failing test**

This is hard to test directly without a real engine. Instead, verify the pattern
in the existing code review — the test here documents the expected behaviour:

Add to `src/__tests__/index.spec.ts`:

```typescript
it('emits an error when position write fails', async () => {
  const uci = new UCI('/invalid/path');
  const errors: Error[] = [];
  uci.on('error', (e) => errors.push(e));

  // Trigger position setter — write will fail because process is dead
  uci.position = 'startpos';

  // Give async error a tick to surface
  await new Promise<void>((resolve) => setTimeout(resolve, 50));

  // At least one error should have been emitted (from uci handshake or position write)
  expect(errors.length).toBeGreaterThan(0);
});
```

**Step 2: Run test to verify current behaviour**


**Step 3: Fix the `position` setter**

In `src/index.ts`, replace:

```typescript
set position(input: string) {
  this.#position = input;
  this.#moves = [];

  this.ready().then(() => this.execute(`position ${input}`));
}
```

With:

```typescript
set position(input: string) {
  this.#position = input;
  this.#moves = [];

  this.ready()
    .then(() => this.execute(`position ${input}`))
    .catch((error: unknown) => {
      void this.emit(
        'error',
        error instanceof Error ? error : new Error(String(error)),
      );
    });
}
```
